# Custom component properties

Omlet provides built-in component properties such as file path, component name, and package name. You can also add your own custom properties to components.

This lets you **define and track custom usage metrics** and **automatically apply tags** based on rules and attributes — giving you a more accurate view of how components are used across your organization.

With custom properties, you can:

- Automatically tag and track components based on their category (Icons, Pages, etc.)
- Automatically tag and track components that are, or should be, deprecated
- Identify all components created by a particular product team or user
- Find components missing documentation or tests
- Differentiate visual vs. non-visual components
- Track package versions used across your projects

Continue with [CLI hooks](./cli-hooks.md) to learn how to add custom properties to components.
